# Hoja de ruta

## Entrega 1: base Apple nativa

Design System, AppShell adaptativo, navegacion, Resumen, Transacciones, estados comunes, previews y pruebas del dominio. Los datos mock permanecen detras de `FinanceRepository`.

## Entrega 2: contratos y datos

OpenAPI, autenticacion de endpoints, repositorios Supabase, dinero decimal y migraciones para bancos, fuentes y estados.

## Entrega 3: cartolas

Carga privada, jobs, extraccion PDF, normalizacion, progreso y bandeja de revision.

## Entrega 4: clasificacion

Edicion, division, por cobrar, transferencias internas y reglas personales.

## Entrega 5: correo

Alias, proveedor inbound, validacion, remitentes bancarios y parsers deterministas.

## Entrega 6: verificacion

Loopback de reenvio con tokens hasheados, expiracion y tres intentos.

## Entrega 7: conciliacion

Fuentes multiples, candidatos, puntuacion, decisiones auditables y duplicados.

## Entregas 8 y 9

Privacy Center, retencion, auditoria, exportacion, notificaciones, observabilidad y optimizacion.

Cada entrega debe incluir migracion reversible, pruebas, compilacion y despliegue independiente.
